# 00. uC/OS-II 嵌入式開發：必備先備知識

這份文件整合了關於 uC/OS-II 專案結構、架構設計與開發環境的核心觀念，是開始進行實驗前的基礎導讀。

---

## 1. 為什麼這個專案看起來很複雜？(心理障礙排除)
如果你來自 Windows 或 Linux 開發背景，可能會感到不適應。原因在於：
*   **黑盒子 vs. 赤裸裸**：一般的 OS 會把核心 (Kernel) 藏起來，你只看得到 App 代碼。但在 uC/OS-II 中，**Kernel 原始碼與你的 App 躺在同一個資料夾**。
*   **上帝兼住戶**：你同時是 App 的開發者 (住戶)，也是系統環境的配置者 (上帝)。

---

## 2. 邊界釐清：核心 (Kernel) vs. 使用者 (User)
為了不被代碼淹沒，我們用表格將「角色與責任」進行明確劃分：

| 類別 | 主要路徑 (資料夾/檔案) | 你的角色 | 它的職責 |
| :--- | :--- | :--- | :--- |
| **核心地帶 (Kernel)** | `uCOS-II\SOURCE` | **觀察者** | 核心大腦 (排程、計時、通信) |
| | `uCOS-II\Ix86L` | **觀察者** | 處理器移植層 (暫存器操作) |
| | `BLOCKS\PC` | **觀察者** | 硬體模擬層 (對應 DOS 功能) |
| | `bc45\` | **使用者** | 編譯工具鏈 (翻譯機器) |
| | `SOFTWARE\TO\` | **使用者** | 目錄跳轉小工具 (傳送門) |
| **使用者地帶 (User)** | `EXn...\SOURCE\TEST.C` | **創造者** | **應用程式邏輯 (你的程式)** |
| | `EXn...\SOURCE\OS_CFG.H`| **上帝 (配置者)** | **系統開關 (控制 OS 功能)** |
| | `EXn...\SOURCE\INCLUDES.H`| **維護者** | 引用管理 (專案標頭檔) |

---

## 3. 作業系統架構：沒有牆的世界
uC/OS-II 採用「**單一地址空間**」與「**平面模式**」，與 Windows/Linux 大不相同：

*   **無特權區隔**：App 與 Kernel 跑在同一個特權等級。App 可以直接呼叫 Kernel 函數。
*   **直接接口**：你不需要 API 抽象層，你是直接連動核心原始碼（例如 `OSSemPost`）。
*   **靜態組合**：編譯時，OS 程式碼會直接「縫合」進你的 App。最後產出的 `.EXE` 是一個包含 OS 功能的單一實體。

---

## 4. 編譯與環境的概念

### 交叉編譯 (Cross-Compilation) vs. 本地編譯 (Native)
*   **從現代電腦視角**：我們在 Windows (64-bit) 幫 DOS (16-bit) 造東西，這是一種 **交叉編譯** 的開發思維。
*   **從 DosBox 視角**：如果你在 DosBox 內執行編譯器，這在術語上稱為 **「本地編譯」**，因為工具與目標環境是一致的。

### 模擬的本質
我們並非「模擬一個硬體」，而是「**利用已知環境當硬體**」。
*   `BLOCKS\PC` 利用 DOS 提供的服務來當作時鐘和顯示器，讓 uC/OS-II 就像跑在真實的嵌入式板子上一樣。

---

## 5. 系統特性：小而強大 (Small Footprint)
你可能會驚訝於 `SOURCE` 資料夾只有 11 個檔案。這是因為 uC/OS-II 是一個 **微內核 (Microkernel)** 架構：

| 特性 | 說明 |
| :--- | :--- |
| **高度精簡** | 它只負責最核心的任務：排程、同步、時間管理。核心碼通常僅幾千行。 |
| **可裁剪性** | 透過 `OS_CFG.H` 關掉不用的功能，編譯後的核心可能只有幾 KB。 |
| **確定性 (Deterministic)** | 因為程式碼少且邏輯清晰，所有操作的執行時間都是可以預測的。 |

### 為什麼它不像 Windows 那麼大？
它剔除了所有非必要的負擔：沒有圖形介面、沒有複雜的驅動資料庫、沒有內建檔案系統。這讓它能跑在極其簡陋的硬體上。

---

## 6. 總結：開發心法
1.  **分層思考**：改程式去 `EXn`，想看底層去 `Ix86`，想看原理去 `SOURCE`。
2.  **直接呼叫**：看到 `OS` 開頭的函數，直接把它當成 Kernel 提供的「服務員」。
3.  **各司其職**：工具歸工具 (`bc45`)，代碼歸代碼 (`SOFTWARE`)。
